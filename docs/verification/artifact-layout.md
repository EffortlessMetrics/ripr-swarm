# Artifact Layout

Adopted repositories use the same paths so humans and agents do not need
repo-specific rituals.

## Committed Public Badge Endpoints

```text
badges/
  README.md
  ripr.json
  ripr-plus.json
  <repo-specific>.json
```

Rules:

- `badges/*.json` contains only Shields endpoint JSON.
- Files are repo-scoped public state.
- Files are generated by `cargo xtask badges`.
- Drift is checked by `cargo xtask badges --check`.
- Detailed reports do not live in `badges/`.

## PR Evidence Artifacts

```text
target/ripr/pr/
  repo-exposure.json
  repo-exposure.md
  summary.md
```

Rules:

- Generated on pull requests.
- Diff-scoped to explicit `base` and `head`.
- Uploaded as CI artifacts.
- Used by job summaries and agents.
- Never used as public badge state.
- `summary.md` is generated from machine-readable PR evidence and review
  guidance artifacts. It is the stable first screen for GitHub summaries.

## Review Guidance Artifacts

```text
target/ripr/review/
  comments.json
  comments.md
  annotations.txt
```

Rules:

- Generated by `cargo xtask ripr-review-comments` or the public
  `ripr review-comments` command.
- `comments[]` is the only annotation source.
- `summary_only[]`, `suppressed[]`, and `warnings[]` stay visible in summaries
  and artifacts.
- The producer does not post to GitHub.
- `annotations.txt` is generated by `cargo xtask ripr-annotations` from
  `comments[]` only. It contains warning-level GitHub Actions annotations.

## Impacted Evidence Artifacts

```text
target/xtask/impacted-evidence/
  latest.json
  latest.md
```

Rules:

- Records the changed files, owners, labels, RIPR gaps, and routing decision.
- Emits machine-readable booleans such as `requires_targeted_mutation` and
  `ripr_severe_gap`.
- Feeds targeted mutation routing.
- Does not execute mutation itself.
- Generated by `cargo xtask impacted-evidence`.

## Existing RIPR Report Packet

RIPR may also produce richer reports under:

```text
target/ripr/reports/
target/ripr/workflow/
target/ripr/receipts/
```

Those paths remain valid for detailed RIPR workflows. The fleet verification
contract standardizes the first-read surfaces for adopted repos:

```text
badges/*.json
target/ripr/pr/*
target/ripr/review/*
target/xtask/impacted-evidence/*
```

When a repo already writes detailed RIPR reports under `target/ripr/reports/`,
its `cargo xtask ripr-pr` wrapper may copy, summarize, or link those reports
into `target/ripr/pr/` instead of re-running analysis.

## Summary Shape

Generated CI summaries should use a stable first screen:

```md
# PR Evidence Summary

## Fast Gate

## RIPR

## Targeted Mutation

## Artifacts
```

Missing data should be shown explicitly. Do not hide missing artifacts behind a
green check or a generic "no issues" message.

Generate the first-screen summary with:

```text
cargo xtask ripr-pr-summary
cargo xtask ripr-pr-summary --check
```
