## Current Work Item

`docs(release): add 0.6.0 release readiness handoff`

0.6.0 is a feature and experience release, not the earlier planned 0.5.1
quality patch. The release line now includes evidence-trust improvements,
policy-operation packets, preview-language visibility, editor first-run
polish, generated-CI and repo-operations front doors, and release-copy
guardrails.

The release remains static and advisory by default:

```text
diff or workspace evidence
-> canonical static evidence
-> editor, CI, policy, and repo-ops packets
-> maintainer judgment or explicit configured gate
```

It must not be described as mutation execution, generated tests, runtime proof,
default blocking, automatic policy mutation, or full multi-language authority.

## Stable, Advisory, And Preview Surfaces

Stable for 0.6.0:

- Rust static mutation-exposure evidence remains the stable product path.
- CLI, JSON, Markdown, editor, generated-CI, and repo-ops packets remain
  source-edit-free and artifact-first.
- `cargo xtask check-product-copy`, `check-pr`, `check-generated-clean`,
  `check-static-language`, and release-readiness checks are release proof
  surfaces.

Advisory by default:

- RIPR findings, generated-CI summaries, policy operations, policy history,
  promotion packets, preview-promotion packets, repo cockpit, PR-ready,
  PR triage, critic, and suggested fixes.
- Calibrated or blocking behavior requires explicit configured policy and the
  already-documented promotion path.

Preview:

- TypeScript, JavaScript, and Python evidence.
- Preview evidence is visible and useful, but not RIPR Zero blocking debt,
  default gate material, mutation-calibrated confidence, or runtime proof.

  expensive mutation testing.

- Editor, CI, and agent surfaces make advisory evidence easier to act on.
- Repo-ops tools make high-volume PR work safer by separating authored truth,
