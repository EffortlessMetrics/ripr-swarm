## Ship Decision

Decision: ready with accepted limitation.

`ripr-swarm` is ready for a source-promotion PR for the 0.7.0 release
candidate. It is not ready for tag or publish from `ripr-swarm`.

The accepted limitation is the current live `ripr-swarm` full-repo audit
timeout. That timeout is bounded, explicit, and fail-closed; it does not
produce fake repair packets. It should be carried as a 0.7.0 limitation unless
a later source-proof PR records a completed live audit and live current-repo
repair proof.

## Release Theme

0.7.0 should be framed as:

```text
RIPR 0.7.0: actionable static repair loop
```

The useful loop is:

```text
existing evidence
-> actionable gap packet
-> ranked swarm plan
-> bounded dry-run repair context
-> focused human or agent edit outside RIPR
-> verify command
-> receipt command
-> actionable-gap outcome
-> next packet or no-action
```

Do not frame 0.7.0 as autonomous repair, generated tests, provider-backed
agents, runtime mutation execution, coverage adequacy, proof of correctness, or
default CI blocking.

## What Is Ready

| Area | Release value |
| --- | --- |
| Actionable gap packets | Existing canonical evidence can be projected into bounded repair packets with canonical gap identity, repair route, target test or assertion shape, verify command, receipt command, confidence basis, supporting raw findings, and must-not-change boundaries. |
| Packet readiness | Actionable packets are audited for structured repair route, verify command, receipt command, confidence basis, and static-limit boundaries before they are treated as repair-ready. |
| Swarm plan | `cargo xtask ripr-swarm plan --top <n>` ranks existing `actionable-gaps.json` packets into swarm-ready and blocked groups without editing files or creating analyzer truth. |
| Dry-run attempt | `cargo xtask ripr-swarm attempt --packet <id> --dry-run` prints one bounded work packet and refuses to become source-edit automation. |
| Human workflow | `docs/RIPR_SWARM_HUMAN_WORKFLOW.md` documents the operator loop from audit through plan, dry-run, repair, verify, receipt, and outcomes. |
| Outcome reporting | `cargo xtask actionable-gap-outcomes` joins packets, receipts, and targeted test outcomes into not-attempted, improved, unchanged, regressed, resolved, attempted-without-receipt, and orphaned-receipt states. |
| Orphan receipt visibility | Stale or unmatched receipts remain visible and do not create or close actionable gaps by themselves. |
| Editor queue | Lane 3 can project the current actionable queue, current repair packet, repo gap map, receipt state, and no-action/fail-closed states from existing typed artifacts. |
| Swarm repository rails | `ripr-swarm` is the active development trunk; source `ripr` remains release/distribution authority; `Ripr Rust Small Result` is the normalized strict PR check. |

## Known Release Limitation

The 0.7 dogfood pass recorded that the current full `ripr-swarm` live audit
timed out before producing current-repo actionable packets:

```text
run limitation: lane1_repo_exposure_timeout
actionable packets emitted: 0
```

This is accepted as a 0.7.0 swarm-repo limitation, not a blocker to source
promotion, because:

- the timeout is bounded and fail-closed;
- incomplete repo-exposure output is not promoted into repair packets;
- the release claim is the actionable static repair loop over existing typed
  artifacts, not a guarantee that the full `ripr-swarm` repository audit
  produces live packets inside the default timeout;
- fixture-backed packet and outcome corpora prove ready, blocked,
  static-limit, missing verify, missing receipt, improved, unchanged, and
  orphaned receipt states;
- source release proof will still run the normal release checks before any tag
  or publish decision.

This limitation becomes a blocker only if the release copy claims:

```text
The current full ripr-swarm audit completes and proves a live repo repair.
```

Do not make that claim for 0.7.0 unless a later source-proof PR adds that
evidence.

## Stable, Advisory, And Preview Surfaces

Stable release boundary:

- source `ripr` remains the release and distribution authority;
- the `ripr` package shape remains one crate and one CLI binary;
- Rust remains the reference evidence path;
- source promotion and source release proof remain mandatory before any public
  0.7.0 publish.

Advisory by default:

- actionable gap packets;
- `ripr-swarm plan`;
- `ripr-swarm attempt --dry-run`;
- actionable-gap outcomes;
- generated CI and PR summaries;
- editor packets and repo gap maps;
- receipts and movement reports unless an explicit later policy gate consumes
  them.

Preview:

- TypeScript, JavaScript, and Python evidence remain opt-in, syntax-first,
  static-limit bounded, advisory, and non-gating by default.

## Public Claims To Make

Use product-facing wording like:

```text
ripr 0.7.0 turns static test-gap evidence into bounded actionable repair
packets: pick one gap, inspect the dry-run context, make a focused human or
agent edit, verify movement, keep the receipt, and review the outcome.
```

Safe public claims:

- `ripr` can rank existing actionable static repair packets.
- `ripr-swarm attempt --dry-run` can print one bounded repair context.
- The repair packet names a repair route, related test or observer, verify
  command, receipt command, confidence basis, and must-not-change boundaries
  when those fields exist.
- Outcome reporting distinguishes not attempted, improved, unchanged,
  regressed, resolved, attempted-without-receipt, and orphaned receipt states.
- The loop is useful for humans and coding-agent operators because it narrows
  the next repair without giving RIPR edit authority.

## Public Claims Not To Make

- Do not claim autonomous repair.
- Do not claim RIPR edits source or generates tests.
- Do not claim provider/model calls.
- Do not claim mutation execution or killed/survived outcomes.
- Do not claim coverage adequacy, runtime adequacy, or correctness proof.
- Do not claim default CI blocking or gate eligibility.
- Do not claim TypeScript, JavaScript, or Python preview evidence is promoted
  to Rust-level confidence or policy authority.
- Do not claim the current full `ripr-swarm` live audit completed and selected
  a live current-repo actionable packet.
